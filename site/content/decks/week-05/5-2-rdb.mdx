<Slide title="What is a database?">

- A database is just organized data
  - Relational – like an excel file (database) with multiple sheets (tables)
  - Key-value – like a folder (database) with multiple files (filename=key, file content = store)
  - Organization based – hierarchical, graph, network

</Slide>

<Slide title="relational database">

- Stores data in tabular form
- Often has multiple tables: different “sets” of data
  - e.g. transactions table, accounts table
- Each table has one or more variables, features, or dimensions as columns
  - Columns are all the same “thing” and have a data type
- Each observation is a row
- Tables within a database are often related somehow

</Slide>

<Slide title="Relational database">

- Related tables “share” information
  - Employee table has demographic information
  - Salary table has pay history for those employees
- Relations often through “keys” that link tables
  - Primary key is used to define an observation (row) in a table
  - Foreign key is included in a table to link to the primary key of another table (link the rows)
  - E.g. Employee ID might be the primary key in Employee, and the foreign key in Salary

</Slide>

<Slide title="Entity relationship diagrams">

- Entities represent the core objects (tables)
- Attributes the specific properties that describe an entity (columns)
- Relationships define how two or more entities are connected to each other (keys)
- Cardinality: specifies how many instances of one entity can be related to how many instances of another entity (m:n)

</Slide>

<Slide title="Relationship Types">

- Relationships can be singular or multiple
  - 1:1 – one row in table 1 matches one row in table 2
  - 1:m – one row in T1 matches multiple rows in T2
    - E.g. 1 employee matches with multiple salary payments
  - m:1 – multiple rows in T1 match with one row in T2
    - E.g. multiple employees will match with 1 manager
  - m:m – multiple rows in T1 match multiple rows in T2
    - E.g. employees could be in multiple committees, and the committees have multiple employees in them

</Slide>
